## When To Use This Skill

Use this for postIntro work: concept proposals, multi-phase choreography,
canonical reference selection, and video-local HTML/CSS/SVG/GSAP implementation
choices.

This skill points to canonical docs rather than copying them. Keep the source
docs authoritative.

## Canonical Docs

- `docs/postintro-patterns.md` — design rules, multi-animation rule, build
  order, canonical references.
- `docs/examples/legacy-postintro-effect-skeleton.md` — default video-local
  legacy/effect skeleton.
- `docs/gsap-flip-patterns.md` — Flip patterns for morphs, reflows, and real UI
  clones.
- `docs/authoring-api.md` — manifest `postIntro` slot and legacy chapter
  contract.

## Required Shape

- 8-15 seconds.
- At least five distinct animation phases.
- At least one cursor or pointer interaction with editorial DOM.
- A clear handoff into the first real product chapter.
- Product-looking UI must be captured, cloned, product-derived, or clearly
  staged as an editorial aid.
